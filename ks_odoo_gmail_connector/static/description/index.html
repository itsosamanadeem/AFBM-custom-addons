Key Features

Send Feature

With Odoo Gmail app, emails can be sent exclusively from the currently logged-in user's email address and are automatically synchronized with their Gmail "Sent" folder. This email functionality offers three convenient ways for sending messages.

Users can send emails directly from various modules such as Sales, Purchase, and Contacts. This feature streamlines communication related to specific records.

Integrated a user-friendly compose menu within Odoo Gmail, that allows users to draft and send emails with ease.

Receive Mail Feature

There are two different ways to receive mails:

Webhook
The webhook will allow the real-time
synchronization of emails received in Google Gmail with Odoo.
Scheduled Action
Emails can be synchronized based on a configured time interval. For example, if the interval is set to 5 minutes, a cron job will run every 5 minutes. During each interval, any relevant emails, such as those matching specified filters, aliases, or existing records, will be synchronized with Odoo.

Gmail Filter Feature

Gmail Filters serve the purpose of filtering the  records of incoming and outgoing emails based on specific criteria, such as keywords like 'Ksolves' found in the subject line of sent and received emails. These filters can be categorized into four distinct types.

Starts with
This filter checks if the specified term is found at the beginning of the subject.
Contains
This filter will sync the email with Odoo if any filter term is found anywhere in the mail Subject.
Ends with
This filter examines whether the provided term appears at the end of the subject.
Regular Expression
This filter operates based on a regular expression pattern match.

Gmail Aliases Feature

When a user configures an email address as an alias, all emails sent or received using that alias will be automatically synchronized with Odoo.

Email Security Feature

Users can access only their email records

Sent Email will be shown in the Sent Menu.



Received Email will be shown in the Inbox Menu.

Setup

Google Application Setup

Step 1: Go to https://console.cloud.google.com/ and click on the select project.

Step 2: After that, create a new project.

Step 3: Change the project name according to the organization requirements and choose your organization.

Step 4: Once the project is created, select that project.

Step 5: Open OAuth consent screen.

Step 6: Create the consent screen; choose the internal type.

Step 7: Go to the credentials menu, select the credentials button, and choose OAuth client ID.

Step 8: Next, Choose Web Application.

Step 9: Enter your website URL, it gives access to your profile login.

Step 10: After creating the OAuth Client ID, you'll receive both the Client ID and Client Secret for use on the Gmail server.

Step 11: Search for the Gmail API and enable it.

Odoo setup

Before Installation: Download the external libraries as mentioned in the requirements.txt file inside the module you downloaded.

Add this configuration parameter in your odoo.conf file to run webhook and you can change values as per your use.

workers=4 (Optional)
server_wide_modules = web,ks_odoo_gmail_connector
[queue_job]
channels = root:4, root.sub_1:2 (Optional)
ODOO_QUEUE_JOB_SCHEME=https
ODOO_QUEUE_JOB_HOST=load-balancer
ODOO_QUEUE_JOB_PORT=443
ODOO_QUEUE_JOB_HTTP_AUTH_USER=jobrunner
ODOO_QUEUE_JOB_HTTP_AUTH_PASSWORD=s3cr3t
ODOO_QUEUE_JOB_JOBRUNNER_DB_HOST=master-db
ODOO_QUEUE_JOB_JOBRUNNER_DB_PORT=5432
longpolling_port = 8072
limit_memory_hard = 13690208256 (As per your requirement)
limit_memory_soft = 11408506880 (As per your requirement)
limit_request = 8192
limit_time_cpu = 120
scheme = https
port = 443
host = your-odoosh-url (For Odoo.Sh Only, and URL without HTTPS protocol)

Step 1: After installing Odoo Gmail Connector, Go to the Users & Companies →Oauth Provider →Open Google OAuth2 record.
Use the client ID that we have created above and the scopes that are provided.

Use the following Scopes:

https://www.googleapis.com/auth/userinfo.email
https://www.googleapis.com/auth/userinfo.profile
https://www.googleapis.com/auth/gmail.compose
https://www.googleapis.com/auth/gmail.readonly
https://www.googleapis.com/auth/gmail.send
https://www.googleapis.com/auth/gmail.labels
https://www.googleapis.com/auth/gmail.modify
UserInfo URL: https://www.googleapis.com/oauth2/v1/tokeninfo
Data URL: https://www.googleapis.com/oauth2/v1/userinfo

Step 2: Go the general setting, select gmail server and fill Client ID and Client Secret of Google cloud console application, which is create in the above setup.

Step 3: Select Webhook or Scheduled action for receiving server.

I. If a user selects Webhook, topic is required.

To get the topic go to the: https://console.cloud.google.com/cloudpubsub

Add principle for all Users permission and select a type pub/sub admin.

After creating a topic for pub/sub, open a subscription.

Next, click the edit button to add your webhook link, where the webhook will be triggered when mail is received.

Add your site URL with the ‘/get mail’ at the end.

Scroll down, Set the expiration to Never Expire.

Next, paste the created topic in the topic field of the Odoo Webhook.

II. If you select the scheduled Action, then a cron job will run every 5 minutes.

Step 4: Assign the Odoo Gmail Connector group to grant the user access to the Gmail menu.
